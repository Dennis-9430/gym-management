Tipos de commit (los más importantes)
Tipo	Uso
feat	Nueva funcionalidad
fix	Corrección de errores
refactor	Cambios internos sin afectar funcionalidad
style	Cambios visuales (CSS, UI, formato)
docs	Documentación
test	Tests
chore	Tareas menores (config, dependencias)
perf	Mejora de rendimiento
ejemplo:
feat(clientes): agregar consumidor final por defecto

descripcion: Explica qué hace el cambio y por qué es necesario.
Opcionalmente agrega detalles técnicos.

## COMANDOS PARA CORRER EL SISTEMA

### Prerrequisitos:
- Python 3.10+
- Node.js 18+
- MongoDB (Docker o local)

### MongoDB (Docker):
docker run -d -p 27017:27017 --name mongodb mongo:latest

### Backend:
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
abrir el servidor

cd backend
uvicorn app.main:app --reload

### Frontend:
npm install
npm run dev

### .env (backend/):
MONGODB_URL=mongodb://localhost:27017
MONGODB_DB_NAME=gym_db
JWT_SECRET_KEY=short-key
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=30
API_HOST=0.0.0.0
API_PORT=8000
DEBUG=True
